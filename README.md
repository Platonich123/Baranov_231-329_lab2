# Лабораторная работа 2
## Тема: Разработка лексического анализатора

### Описание
В рамках лабораторной работы разработан модуль лексического анализатора для подмножества языка **C**, используемого в тестовой программе из лабораторной работы 1.

Программа принимает на вход очищенный файл `test_cleaned.c`, выполняет разбор текста на лексемы, определяет тип каждой лексемы, формирует таблицы лексем и выводит последовательность токенов, пригодную для передачи на следующий этап компиляции.

### Поддерживаемые типы лексем
- ключевые слова;
- идентификаторы;
- целочисленные константы;
- вещественные константы;
- строковые константы;
- булевы константы;
- операторы;
- разделители.

### Особенности реализации
- поддерживается разбор конструкций, реально встречающихся в `test_cleaned.c`;
- корректно обрабатываются строки с escape-последовательностями, например `"...\n"`;
- поддерживаются составные операторы `<=`, `==`, `&&`, `++`;
- предусмотрена минимальная поддержка директив препроцессора `#include`;
- обнаруживаются лексические ошибки:
  - недопустимые символы;
  - идентификаторы, начинающиеся с цифры;
  - некорректно оформленные числа;
  - незакрытые строковые литералы.

### Состав проекта
- `lexer.py` — модуль лексического анализатора;
- `test_cleaned.c` — очищенный входной файл из лабораторной работы 1;
- `example_console_output.txt` — пример успешной работы программы;
- `output/lexeme_tables.txt` — таблицы лексем;
- `output/token_sequence.txt` — таблица последовательности токенов;
- `output/token_pairs.txt` — список пар `(TYPE, value)`;
- `report/report_lab2_draft.md` — черновик текста отчёта.

### Запуск
```bash
python lexer.py test_cleaned.c
```

Для сохранения файлов, которые можно приложить к отчёту:
```bash
python lexer.py test_cleaned.c output
```

### Пример
После запуска программа:
1. читает `test_cleaned.c`;
2. строит таблицы лексем;
3. печатает последовательность токенов;
4. выводит список вида `(TYPE, value)`;
5. сообщает число найденных токенов и ошибки, если они есть.
